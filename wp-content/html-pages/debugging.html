Defining Debugging Constants

We can define WP_DEBUG constants and read the debug logs.

Enable debugging in the Playground

To allow debugging, run wp-now start --blueprint=path/to/debug-wp-config-example.json where debug-wp-config-example.json

{
	"steps": [
		{
			"step": "defineWpConfigConsts",
			"consts": {
				"WP_DEBUG": true,
				"WP_DEBUG_LOG": true
			},
			"method": "define-before-run"
		}
	]
}

This will enable the debug logs and will create a debug.log file in the ~/.wp-now/wp-content/${project}/debug.log directory.

Known Limitations

Errors may occur on the front-end when defining constants in this way, because they are already defined by WordPress. The debugging will still work, it just displays an error.

wp-now: Constant WP_DEBUG already defined

See #135 for more context

Workaround

To utilize the default wp-config values and stop the error message, you can change method to rewrite-wp-config ie:

 wp-now start --blueprint=path/to/debug-wp-config-append-example.json 

{
	"steps": [
		{
			"step": "defineWpConfigConsts",
			"consts": {
				"WP_DEBUG": true,
				"WP_DEBUG_LOG": true
			},
			"method": "rewrite-wp-config "
		}
	]
}
